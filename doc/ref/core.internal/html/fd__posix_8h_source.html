64  /* The watcher list.
65 
66  The following watcher related fields are protected by watcher_mu.
67 
68  An fd_watcher is an ephemeral object created when an fd wants to
69  begin polling, and destroyed after the poll.
70 
71  It denotes the fd's interest in whether to read poll or write poll
72  or both or neither on this fd.
73 
74  If a watcher is asked to poll for reads or writes, the read_watcher
75  or write_watcher fields are set respectively. A watcher may be asked
76  to poll for both, in which case both fields will be set.
77 
78  read_watcher and write_watcher may be NULL if no watcher has been
79  asked to poll for reads or writes.
80 
81  If an fd_watcher is not asked to poll for reads or writes, it's added
82  to a linked list of inactive watchers, rooted at inactive_watcher_root.
83  If at a later time there becomes need of a poller to poll, one of
84  the inactive pollers may be kicked out of their poll loops to take
85  that responsibility. */

115 /* Begin polling on an fd.
116  Registers that the given pollset is interested in this fd - so that if read
117  or writability interest changes, the pollset can be kicked to pick up that
118  new interest.
119  Return value is:
120  (fd_needs_read? read_mask : 0) | (fd_needs_write? write_mask : 0)
121  i.e. a combination of read_mask and write_mask determined by the fd's current
